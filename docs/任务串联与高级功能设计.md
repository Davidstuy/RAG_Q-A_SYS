### 1. 任务编排器
```java
@Component
@RequiredArgsConstructor
public class TaskOrchestrator {
    
    private final ToolRegistry toolRegistry;
    private final DialogueManager dialogueManager;
    private final ChatLanguageModel chatModel;
    
    /**
     * 执行复杂任务
     */
    public TaskResult executeComplexTask(String sessionId, String userInput) {
        // 1. 任务分解
        List<TaskStep> steps = decomposeTask(userInput);
        
        // 2. 逐步执行
        Map<String, Object> results = new HashMap<>();
        for (TaskStep step : steps) {
            TaskStepResult stepResult = executeStep(step, sessionId, results);
            results.put(step.getId(), stepResult);
            
            if (!stepResult.isSuccess()) {
                return TaskResult.failed("任务执行失败: " + stepResult.getErrorMessage());
            }
        }
        
        // 3. 整合结果
        String finalResult = integrateResults(steps, results);
        return TaskResult.success(finalResult);
    }
    
    private List<TaskStep> decomposeTask(String userInput) {
        // 使用 LLM 分析复杂任务并分解
        String prompt = String.format("""
            请将用户的复杂任务分解为多个步骤。
            
            用户输入: %s
            
            请返回 JSON 格式的步骤列表:
            [
                {
                    "id": "step1",
                    "description": "第一步描述",
                    "tool": "工具名称",
                    "parameters": {
                        "param1": "value1"
                    },
                    "depends_on": []
                }
            ]
            """, userInput);
        
        String response = chatModel.generate(prompt);
        return parseTaskSteps(response);
    }
    
    private TaskStepResult executeStep(TaskStep step, String sessionId, Map<String, Object> previousResults) {
        try {
            // 处理依赖参数
            Map<String, Object> parameters = resolveParameters(step.getParameters(), previousResults);
            
            // 执行工具
            Tool tool = toolRegistry.getTool(step.getTool());
            if (tool == null) {
                return TaskStepResult.failed("工具不存在: " + step.getTool());
            }
            
            ToolInput input = new ToolInput();
            input.setParameters(parameters);
            input.setSessionId(sessionId);
            
            ToolExecution result = tool.execute(input);
            return TaskStepResult.success(result.getResult());
            
        } catch (Exception e) {
            return TaskStepResult.failed("步骤执行失败: " + e.getMessage());
        }
    }
    
    private Map<String, Object> resolveParameters(Map<String, Object> parameters, Map<String, Object> previousResults) {
        Map<String, Object> resolved = new HashMap<>();
        
        for (Map.Entry<String, Object> entry : parameters.entrySet()) {
            String value = (String) entry.getValue();
            
            // 如果参数引用了前面的步骤结果
            if (value.startsWith("$")) {
                String stepId = value.substring(1);
                resolved.put(entry.getKey(), previousResults.get(stepId));
            } else {
                resolved.put(entry.getKey(), value);
            }
        }
        
        return resolved;
    }
    
    private String integrateResults(List<TaskStep> steps, Map<String, Object> results) {
        // 使用 LLM 整合所有步骤的结果
        StringBuilder context = new StringBuilder();
        for (TaskStep step : steps) {
            context.append(step.getDescription()).append(": ")
                   .append(results.get(step.getId())).append("\n");
        }
        
        String prompt = String.format("""
            请整合以下任务执行结果，生成一个完整的回答。
            
            执行过程:
            %s
            
            请生成一个连贯的回答。
            """, context.toString());
        
        return chatModel.generate(prompt);
    }
}
```

### 2. 任务步骤定义
```java
@Data
public class TaskStep {
    private String id;
    private String description;
    private String tool;
    private Map<String, Object> parameters;
    private List<String> dependsOn; // 依赖的步骤ID
    
    public boolean isReady(Map<String, Object> completedSteps) {
        return dependsOn.stream().allMatch(completedSteps::containsKey);
    }
}

@Data
@AllArgsConstructor
public class TaskStepResult {
    private boolean success;
    private Object result;
    private String errorMessage;
    
    public static TaskStepResult success(Object result) {
        return new TaskStepResult(true, result, null);
    }
    
    public static TaskStepResult failed(String errorMessage) {
        return new TaskStepResult(false, null, errorMessage);
    }
}

@Data
@AllArgsConstructor
public class TaskResult {
    private boolean success;
    private String result;
    private String errorMessage;
    
    public static TaskResult success(String result) {
        return new TaskResult(true, result, null);
    }
    
    public static TaskResult failed(String errorMessage) {
        return new TaskResult(false, null, errorMessage);
    }
}
```

### 1. 自动重试机制
```java
@Component
public class RetryExecutor {
    
    private static final int MAX_RETRIES = 3;
    private static final long RETRY_DELAY_MS = 1000;
    
    public <T> T executeWithRetry(Callable<T> task, String taskName) {
        int attempt = 0;
        while (attempt < MAX_RETRIES) {
            try {
                return task.call();
            } catch (Exception e) {
                attempt++;
                if (attempt >= MAX_RETRIES) {
                    throw new RuntimeException("任务执行失败: " + taskName, e);
                }
                
                try {
                    Thread.sleep(RETRY_DELAY_MS * attempt);
                } catch (InterruptedException ie) {
                    Thread.currentThread().interrupt();
                    throw new RuntimeException("重试被中断", ie);
                }
            }
        }
        throw new RuntimeException("不应该到达这里");
    }
}
```

```java
public class ToolCache {

        return cache.get(key, k -> loader.get());

    public void invalidate(String key) {
        cache.invalidate(key);
    }
    
    @Data
    private static class CacheEntry {

    }
}
```

### 3. 错误处理和恢复
```java
@Component
public class ErrorHandler {
    
    private final DialogueManager dialogueManager;
    
    public ErrorRecoveryResult handleError(Exception e, String sessionId) {
        ErrorType errorType = classifyError(e);
        
        switch (errorType) {
            case NETWORK_ERROR:
                return handleNetworkError(e);
            case API_RATE_LIMIT:
                return handleRateLimitError(e);
            case INVALID_PARAMETER:
                return handleParameterError(e);
            case TOOL_NOT_FOUND:
                return handleToolNotFoundError(e);
            default:
                return handleGenericError(e);
        }
    }
    
    private ErrorType classifyError(Exception e) {
        if (e instanceof NetworkException) {
            return ErrorType.NETWORK_ERROR;
        } else if (e instanceof RateLimitException) {
            return ErrorType.API_RATE_LIMIT;
        } else if (e instanceof ParameterException) {
            return ErrorType.INVALID_PARAMETER;
        } else if (e instanceof ToolNotFoundException) {
            return ErrorType.TOOL_NOT_FOUND;
        } else {
            return ErrorType.UNKNOWN;
        }
    }
    
    private ErrorRecoveryResult handleNetworkError(Exception e) {
        // 网络错误：重试或使用缓存
        return ErrorRecoveryResult.retry("网络连接异常，正在重试...");
    }
    
    private ErrorRecoveryResult handleRateLimitError(Exception e) {
        // API 限流：等待后重试
        return ErrorRecoveryResult.waitAndRetry("API 调用过于频繁，请稍后重试");
    }
    
    private ErrorRecoveryResult handleParameterError(Exception e) {
        // 参数错误：请求用户提供正确参数
        return ErrorRecoveryResult.requestParameter(e.getMessage());
    }
    
    private ErrorRecoveryResult handleToolNotFoundError(Exception e) {
        // 工具不存在：建议替代方案
        return ErrorRecoveryResult.suggestAlternative("抱歉，暂时无法处理这个请求，您可以尝试其他方式");
    }
    
    private ErrorRecoveryResult handleGenericError(Exception e) {
        // 通用错误：记录日志并返回友好提示
        return ErrorRecoveryResult.error("抱歉，处理您的请求时出现了问题，请稍后重试");
    }
}

@Data
@AllArgsConstructor
public class ErrorRecoveryResult {
    private RecoveryAction action;
    private String message;
    private Map<String, Object> data;
    
    public static ErrorRecoveryResult retry(String message) {
        return new ErrorRecoveryResult(RecoveryAction.RETRY, message, null);
    }
    
    public static ErrorRecoveryResult waitAndRetry(String message) {
        return new ErrorRecoveryResult(RecoveryAction.WAIT_AND_RETRY, message, null);
    }
    
    public static ErrorRecoveryResult requestParameter(String message) {
        return new ErrorRecoveryResult(RecoveryAction.REQUEST_PARAMETER, message, null);
    }
    
    public static ErrorRecoveryResult suggestAlternative(String message) {
        return new ErrorRecoveryResult(RecoveryAction.SUGGEST_ALTERNATIVE, message, null);
    }
    
    public static ErrorRecoveryResult error(String message) {
        return new ErrorRecoveryResult(RecoveryAction.ERROR, message, null);
    }
}

public enum RecoveryAction {
    RETRY,
    WAIT_AND_RETRY,
    REQUEST_PARAMETER,
    SUGGEST_ALTERNATIVE,
    ERROR
}
```

### 示例1: 复杂任务执行
```
用户: "帮我查一下北京的天气，如果温度超过25度，提醒我注意防暑，然后推荐一些适合的运动"

任务分解:
1. 查询北京天气 -> WeatherTool
2. 判断温度是否超过25度 -> 条件判断
3. 如果超过25度，生成防暑提醒 -> 提醒生成
4. 推荐适合的运动 -> 运动推荐工具

执行流程:
1. 执行天气查询: 北京天气：晴，温度28°C
2. 判断: 28 > 25，需要提醒
3. 生成提醒: "北京今天温度较高(28°C)，请注意防暑，多喝水，避免长时间户外活动"

5. 整合结果: 完整回答
```

### 1. 更新 pom.xml
```xml

<dependency>
    <groupId>dev.langchain4j</groupId>
    <artifactId>langchain4j-agent</artifactId>
    <version>0.29.1</version>
</dependency>

<dependency>
    <groupId>com.github.ben-manes.caffeine</groupId>
    <artifactId>caffeine</artifactId>
    <version>3.1.8</version>
</dependency>

<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-cache</artifactId>
</dependency>
```

### 2. 配置 application.yml
```yaml
agent:
  max-retry-attempts: 3
  retry-delay-ms: 1000
  cache-ttl: 3600000
  tools:
    weather:
      enabled: true
      api-key: ${WEATHER_API_KEY}
    currency:
      enabled: true
      api-key: ${CURRENCY_API_KEY}
```

### 3. 启用缓存
```java
@SpringBootApplication
@EnableCaching

    public static void main(String[] args) {
        SpringApplication.run(AgentApplication.class, args);
    }
}
```
